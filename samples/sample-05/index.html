Sample 05 · Capstone

Personal Finance Copilot

No visibility into spending until month end — solved by categorising bank exports in Claude, building a Q&A session over your own data, and generating a monthly roll-up Artifact.

No API key needed. This capstone runs entirely inside your Claude.ai subscription — transaction data pasted directly into Claude (never sent to a third-party API), analysis prompts saved to your library, monthly report assembled as an Artifact. No Python required.

The Problem & Solution Overview

The problem: You only realise you overspent on restaurants at month end, when it's too late to adjust. Bank apps give you raw transaction lists but no meaningful categories, no budget tracking, and no natural-language way to ask questions like "am I on track for my grocery budget?"

The solution: A personal finance copilot built entirely in Claude.ai. You paste your bank export CSV into a project conversation, Claude categorises every transaction using a consistent prompt, and you ask natural-language questions about your spending. A monthly roll-up prompt produces a clean summary Artifact you can review in minutes.

CSV Ingest + Prompt (HO2) Grounded Q&A (HO4) Monthly Roll-up Automation (HO5-6)
Privacy note: When using real bank data, do NOT paste account numbers, sort codes, or full card numbers. Most bank exports include only merchant name, date, and amount — that is all you need. Anonymise or use the sample data in this repo when practising.

Component 1

Grounding — Budget Rules Claude Project

Load your budget rules and category definitions into a Claude Project once. Every monthly session uses the same rules automatically — no re-explaining your budget each time.

Steps

  1. Go to claude.ai → Projects → New Project. Name it "Finance Copilot".
  2. Add Project Knowledge: paste the contents of components/02-context-pack/financial-rules.txt. Title it "Budget Rules & Categories". This file contains your category definitions and monthly budget targets.
  3. Set the Project System Prompt: "You are a personal finance assistant. Use the Budget Rules & Categories document to define spending categories and budget targets. Always cite the relevant budget rule when assessing whether spending is on-track. Never guess at categories — if a merchant doesn't clearly fit a category, use 'uncategorised'."
  4. Customise the financial-rules.txt before uploading: add your actual monthly budget targets per category (e.g. Groceries: £300/month, Restaurants: £150/month).

Sample data

The file components/03-automation/sample_transactions.csv contains 23 anonymised sample transactions from October 2024 — use these to practise before adding real bank data.

Component 2

Prompt Library — Three Finance Prompts

Save each prompt to your Claude prompt library. Run them in sequence each month inside the Finance Copilot project.

Prompt 1: Transaction Categoriser

Save as: "Finance — Categorise Transactions"

Using the Budget Rules & Categories in this project, categorise every transaction in the CSV data below. For each transaction, assign: 1. Category (from the categories defined in the project knowledge) 2. Confidence (high / medium / low) 3. Note (brief reason if confidence is medium or low) Output as a table: | Date | Merchant | Amount | Category | Confidence | Note | |------|----------|--------|----------|------------|------| After the table, list any transactions you marked "uncategorised" and suggest a likely category with a question for me to confirm. CSV data: --- [PASTE YOUR CSV HERE — Date, Merchant, Amount columns only] ---

Prompt 2: Spending Q&A

Save as: "Finance — Q&A Mode"

Based on the categorised transaction table above and the budget targets in the Budget Rules & Categories document, answer my questions about my spending. For each question: - Give a direct numerical answer - Compare to my budget target if one exists for that category - Flag if I am over budget (use "OVER BUDGET" in bold) - Suggest one specific action if I am over budget I'll ask my questions one at a time. Start by confirming: how many transactions are in the data, the date range they cover, and the total spend. Ready? I'll ask my first question now.

Prompt 3: Monthly Roll-up

Save as: "Finance — Monthly Roll-up"

Using all the categorised transaction data and Q&A from this conversation, produce a monthly financial summary. Format as a Claude Artifact: # Monthly Finance Report — [Month Year] ## Summary - Total spend: £[amount] - Transactions analysed: [count] - Date range: [start] to [end] ## Spend by Category | Category | Amount | Budget | Status | |----------|--------|--------|--------| [one row per category — Status = On Track / Over Budget / Under Budget] ## Top 5 Merchants [ranked list with total spend per merchant] ## Budget Performance [One paragraph: which categories were on-track, which were over, overall assessment] ## Recommendations for Next Month [Numbered list: 3 specific, actionable suggestions based on the data] ## Transactions Needing Review [List any uncategorised or low-confidence items that need manual check] Base every number on the categorised data above. Do not estimate or infer amounts not present in the data.
Component 3

Automation — Monthly Finance Session

Run one Claude session per month. The three prompts handle the full workflow — from raw bank export to a shareable monthly report — in under 20 minutes.

Monthly workflow

  1. Export your bank transactions for the month as CSV. Keep only: Date, Merchant Name, Amount. Remove any account numbers or card details.
  2. Open your Finance Copilot Claude Project. Start a new conversation titled "Finance — [Month Year]".
  3. Apply the "Finance — Categorise Transactions" prompt. Paste your CSV in place of the placeholder. Review the output table — confirm any "uncategorised" items Claude asks about.
  4. Apply the "Finance — Q&A Mode" prompt to enter interactive mode. Ask your spending questions one at a time. Example questions:
    • "How much did I spend on restaurants?"
    • "Am I on track for my grocery budget?"
    • "What were my three biggest individual transactions?"
    • "How does this month compare to my budget overall?"
  5. Apply the "Finance — Monthly Roll-up" prompt. Claude produces a formatted Artifact summarising the month.
  6. Save or share the Artifact. Archive the conversation. Start fresh next month with a new conversation in the same project.

Putting It Together — End-to-End Finance Workflow

Complete monthly finance review workflow using only Claude.ai:

  1. Month end (5 min setup): Export bank CSV → strip to Date/Merchant/Amount → open Finance Copilot project → start new conversation.
  2. Categorise (5 min): Apply Categorise Transactions prompt → review table → confirm any uncategorised items.
  3. Q&A (5–10 min): Apply Q&A Mode prompt → ask your 3–5 most important spending questions → note anything surprising.
  4. Report (2 min): Apply Monthly Roll-up prompt → Artifact appears → review the recommendations.
  5. Action: Pick one recommendation to act on next month. Update your financial-rules.txt budget targets if needed (Project Knowledge → edit document).

Total time per month: approximately 15–20 minutes. No spreadsheets, no finance apps, no third-party data sharing.
