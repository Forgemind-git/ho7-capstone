AI Support Triage System

Support team buried in tickets with no trends view — solved with Claude Projects, a Prompt Library, and a Cowork automation workflow.

No API key needed. This capstone runs entirely inside your Claude.ai subscription — using Claude Projects for grounding, saved prompts for the library, and Cowork for automation. No Python required.

The Problem & Solution Overview

The problem: Your support team receives dozens of tickets daily. Each agent decides independently how to categorise a ticket, what severity to assign, and how to draft a reply — leading to inconsistent responses, missed critical bugs, and no way to spot trends until it's too late.

The solution: A three-component triage system built entirely in Claude.ai. A Claude Project holds all your product documentation as grounding context, a prompt library ensures every ticket is classified the same way, and a Cowork workflow processes batches of tickets end-to-end in a single session.

Prompt Library (HO2) Grounded Context (HO4) Workflow Automation (HO5-6)

When complete, any team member opens one Claude Project and can tag, score, and draft responses for a batch of tickets in under 10 minutes — with consistent, grounded output every time.

Component 1

Grounding — Set Up Your Claude Project

A Claude Project makes your product documentation permanently available as grounding context. Every conversation inside the project automatically has access to the docs — you never paste them in manually.

Steps

  1. Go to claude.ai → Projects → New Project. Name it "Support Triage System".
  2. In Project Knowledge, click Add content and paste the contents of components/02-context-pack/product-docs.txt from this repo (or your own product documentation).
  3. Add a second knowledge document: paste the contents of components/02-context-pack/known-issues.txt. Title it "Known Issues Index".
  4. Set a Project System Prompt (optional but recommended): "You are a support triage assistant. Always cite the relevant documentation section when drafting ticket responses. Use the known issues index to check if a reported problem is already tracked."
  5. Save the project. Every new conversation in this project now has the docs loaded.

What to verify

Start a conversation inside the project and ask: "What is the password reset flow?" — Claude should cite Section 1.1 of your product docs without you providing any context. If it does, grounding is working.

Component 2

Prompt Library — Build the Three Triage Prompts

Save each prompt below to your Claude prompt library (claude.ai → Prompts → New Prompt). Give each a clear name so you can recall them quickly during triage.

Prompt 1: Ticket Tagger

Save as: "Support — Tag Ticket"

You are a support ticket classifier for a B2B SaaS product. Assign ONE category tag to the ticket below. Available tags: - authentication (login, password reset, SSO, 2FA) - billing (invoices, subscription, payment failure, refunds) - mobile-bug (crash or malfunction on iOS or Android) - web-bug (crash or malfunction on web/browser) - performance (slow load times, timeouts, lag) - data-export (CSV/PDF/API export issues) - integration (third-party connectors: Slack, Zapier, Salesforce) - feature-request (user wants a new capability) - account-management (user/seat/role management) - other (does not fit any category above) Return ONLY a JSON object: {"tag": "<tag>", "confidence": <0.0-1.0>} Ticket: [PASTE TICKET SUBJECT AND BODY HERE]

Prompt 2: Severity Scorer

Save as: "Support — Score Severity"

You are a support triage specialist. Assign a severity level (1–4) to this ticket. Severity scale: 1 = Critical — service down, data loss possible, immediate response needed 2 = High — key feature broken, no workaround, respond within 4 hours 3 = Medium — feature partially working, workaround exists, respond within 1 day 4 = Low — minor issue or informational request, respond within 3 days Return ONLY a JSON object: {"severity": <1|2|3|4>, "rationale": "<one sentence>"} Tag already assigned: [TAG FROM STEP 1] Ticket: [PASTE TICKET SUBJECT AND BODY HERE]

Prompt 3: Response Drafter

Save as: "Support — Draft Response"

You are a support agent drafting a reply to a customer ticket. Use the product documentation loaded in this project to write a helpful, grounded response. Rules: - Be concise and empathetic (2–4 short paragraphs) - Cite the specific documentation section that applies (e.g. "As described in Section 1.1...") - If the issue is in the Known Issues Index, mention it is tracked and give an ETA if available - End with a clear next step for the customer Ticket category: [TAG] Severity: [SEVERITY LEVEL] Ticket: [PASTE TICKET SUBJECT AND BODY HERE]
Component 3

Automation — Cowork Batch Triage Workflow

Use Claude's Cowork feature to process multiple tickets in sequence within your Support Triage Project. This replaces the Python automation script — no code needed.

Steps

  1. Open your Support Triage System Claude Project and start a new conversation.
  2. Paste your batch of tickets at the top of the conversation. Use this structure for each ticket:
--- TICKET 001 ---
Subject: Cannot log in after password reset
Body: Hi, I reset my password this morning but the link had already expired by the time I clicked it. I'm now locked out. Please help urgently.
  1. Use Prompt 1 (Ticket Tagger) on each ticket. Type: "Apply the Support — Tag Ticket prompt to TICKET 001". Repeat for each ticket.
  2. Once all tickets are tagged, use Prompt 2 (Severity Scorer) for each: "Apply Support — Score Severity to TICKET 001. Tag is: authentication".
  3. For tickets with Severity 1 or 2, use Prompt 3 (Response Drafter): "Draft a response for TICKET 001. Tag: authentication. Severity: 2."
  4. At the end of the session, ask Claude: "Summarise the tickets we processed today. List: total count, breakdown by tag, count of severity 1 and 2 tickets, and any patterns in the issues." — this is your trend report.

Tip: Use an Artifact for tracking

Ask Claude to create an Artifact table at the start: "Create an Artifact table with columns: Ticket ID, Tag, Severity, Action Required." Update it after each ticket — you can export it as your triage log.

Putting It Together — End-to-End Workflow

Here is the complete workflow a support agent follows each morning using only Claude.ai:

  1. Open the Support Triage System Claude Project.
  2. Paste all new tickets from your help desk (export as text or CSV, paste the relevant columns).
  3. Tag each ticket using the "Support — Tag Ticket" prompt from your library.
  4. Score each ticket using the "Support — Score Severity" prompt. Any Severity 1 tickets — escalate immediately.
  5. Draft responses for Severity 1 and 2 tickets using the "Support — Draft Response" prompt. Claude uses the project docs automatically — no copy-pasting context.
  6. Generate the weekly trend summary by asking Claude to aggregate all tickets from the session (or paste a batch from the week).
  7. Export the Artifact table as your triage log. Share the trend summary with your team lead.
