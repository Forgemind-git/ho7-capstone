Content Production Engine

Content team cannot scale research and production — solved with a brand-grounded Claude Project, a multi-format prompt library, and a Cowork pipeline that takes one topic brief to five content pieces.

No API key needed. This capstone runs entirely inside your Claude.ai subscription — brand voice and audience personas loaded as Project Knowledge, drafting prompts saved to your library, full content bundle assembled via Cowork. No Python required.

The Problem & Solution Overview

The problem: A single content piece takes 3–4 hours from brief to publish: research, drafting, editing for brand voice, and then repurposing the same idea into LinkedIn posts, Twitter threads, and email newsletters. The team spends more time on reformatting than on original thinking.

The solution: A Claude-powered content production pipeline. Brand voice, audience personas, and trend signals live permanently in a Claude Project. Three prompts handle research brief → blog draft → multi-format repurposing. One Cowork session produces five content pieces in under 30 minutes.

Trend Research / Grounding (HO4) Prompt Library (HO2) Pipeline Automation (HO5-6)

Output: a research brief, a full blog post, a LinkedIn article, a Twitter/X thread, and an email newsletter — all on-brand, grounded in audience data, produced from a single topic sentence.

Every output automatically matches your brand voice because the brand voice document is always in the project context. Interviewers never need to paste "write in our brand voice" — it's always there.

Component 1

Research Grounding — Load Brand & Audience Context

Load your brand and audience materials into a Claude Project once. Every content piece produced in the project is automatically grounded in your voice and audience — no copy-pasting guidelines into each conversation.

Steps

  1. Go to claude.ai → Projects → New Project. Name it "Content Engine".
  2. Add Project Knowledge document 1: paste components/02-context-pack/brand-voice.txt. Title it "Brand Voice Guide".
  3. Add document 2: paste components/02-context-pack/audience-personas.txt. Title it "Audience Personas".
  4. Add document 3: paste components/02-context-pack/trend-signals.txt. Title it "Trend Signals — [Month Year]".
  5. Set the Project System Prompt: "You are a content strategist and writer. Always write in the brand voice defined in the Brand Voice Guide. Always consider the audience personas before structuring content. Use the Trend Signals document as research context — cite relevant trends when appropriate."
  6. Update the Trend Signals document monthly to keep research current.

Verification

Ask Claude: "Describe our brand voice in three adjectives." — the answer should come directly from your brand-voice document. If it describes a generic voice instead, check the document was saved correctly in Project Knowledge.

Component 2

Prompt Library — Three Production Prompts

Save each prompt to your Claude prompt library. The three prompts form a sequential pipeline — run them in order in each content session.

Prompt 1: Research Brief

Save as: "Content — Research Brief"

Using the Trend Signals and Audience Personas in this project, create a structured research brief for the following topic: Topic: [PASTE TOPIC BRIEF HERE] Output a Research Brief with these sections: ## Core Insight [One sentence: the most important thing the audience needs to understand] ## Audience Pain Point [Which persona is this for? What pain point does this topic address? Cite the personas document.] ## Trend Angle [Which trend signal makes this topic timely right now? Direct quote or reference from Trend Signals document.] ## Key Points to Cover [Bullet list: 4–6 must-include points for a complete treatment of this topic] ## Content Angle [One sentence: the unique angle that makes our take different from generic content on this topic] ## Suggested Headline Options [3 headline options — vary format: question / statement / how-to]

Prompt 2: Blog Draft

Save as: "Content — Blog Draft"

Using the Research Brief above and the Brand Voice Guide in this project, write a full blog post. Requirements: - Length: 1,000–1,500 words - Structure: introduction, 3–4 main sections with subheadings, conclusion with CTA - Voice: match exactly the Brand Voice Guide in this project - Audience: write for the primary persona identified in the Research Brief - Include the trend angle naturally within the body — do not force it Use one of the headline options from the Research Brief (or improve on them). Format as a Claude Artifact so I can copy it directly.

Prompt 3: Repurpose

Save as: "Content — Repurpose"

Repurpose the blog post above into [FORMAT] format. Formats and their requirements: If FORMAT = LinkedIn post: - 150–300 words - Opening hook in the first line (no "I" start) - 3–5 short paragraphs or bullet points - Conversational, first-person brand voice - End with a question to prompt comments If FORMAT = Twitter/X thread: - 6–10 tweets - Tweet 1: bold hook (no hashtags) - Tweets 2–8: one key point per tweet, self-contained - Final tweet: summary + follow CTA - Each tweet under 280 characters If FORMAT = Email newsletter: - Subject line + preview text - 250–400 words body - One clear CTA at the end - Friendly, slightly more personal than blog voice Apply this prompt three times (one per format) and create each as a separate Artifact. Current format requested: [PASTE FORMAT HERE]
Component 3

Automation — Cowork Content Pipeline Session

Run one Cowork session per topic to produce the full content bundle. A complete session takes 20–30 minutes and produces five on-brand content pieces.

Session workflow

  1. Open the Content Engine Claude Project. Start a new conversation titled with the topic (e.g. "Content — Async Teams — 28 June 2025").
  2. Apply the "Content — Research Brief" prompt. Paste your topic sentence or brief. Review the output — adjust the angle or pain point if needed before proceeding.
  3. Apply the "Content — Blog Draft" prompt. Read the draft, make any edits inline (ask Claude to revise specific sections if needed).
  4. Apply the "Content — Repurpose" prompt three times — once per format. Change the format line each time: "LinkedIn post", "Twitter/X thread", "Email newsletter".
  5. You now have four Artifacts in the conversation (blog + three formats). Export them to your content calendar tool (Notion, Airtable, Buffer draft).

Topic brief format

You can use the sample topic brief from the repo (components/03-automation/topic_brief.txt) as a template. A good brief is 1–3 sentences: the topic, the target audience, and the specific angle or question to address.

Putting It Together — End-to-End Content Workflow

Complete weekly content workflow for a content team using Claude.ai:

  1. Monday — topic selection: Choose 1–3 topics for the week. Write a one-sentence brief for each.
  2. For each topic — research: Open Content Engine project → apply Research Brief prompt → review brief, refine angle if needed.
  3. For each topic — draft: Apply Blog Draft prompt → review and edit the draft → approve.
  4. For each topic — repurpose: Apply Repurpose prompt three times (LinkedIn / Twitter / Email) → you have a full content bundle.
  5. Publish: Copy Artifacts to your publishing tools. One session in Claude = a full week of social + email content.
  6. Monthly update: Refresh the Trend Signals document in Project Knowledge with new data. The next session automatically incorporates updated trends.
