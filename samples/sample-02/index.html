Competitor Intelligence Tracker

Marketing team cannot keep up with competitor releases — solved with a Claude Project research grounding, analysis prompt library, and Cowork-driven report generation.

No API key needed. This capstone runs entirely inside your Claude.ai subscription — competitor materials uploaded as Project Knowledge, analysis prompts saved to your library, Cowork handles the report assembly. No Python required.

The Problem & Solution Overview

The problem: Your product and marketing teams need to know what competitors are shipping — pricing changes, new features, messaging pivots — but tracking multiple competitors manually takes hours each week, and the output is usually stale by the time it reaches the team.

The solution: A structured intelligence pipeline in Claude.ai. Competitor release notes, feature pages, and pricing info are loaded into a Claude Project as grounding context. Three analysis prompts extract gaps, compare pricing, and identify messaging opportunities. A Cowork session assembles the final shareable report.

Prompt Library (HO2) Grounded Research (HO4) Skill-Driven Automation (HO5-6)

Result: a complete gap analysis report — feature table, pricing comparison, messaging opportunities, and recommended actions — produced in a single Claude session, grounded in real competitor data.

Component 1

Research Grounding — Load Competitor Materials into Claude Project

Rather than pasting competitor data into every chat, load it once into a Claude Project. The project makes all materials available as permanent grounding context across every conversation.

Steps

  1. Go to claude.ai → Projects → New Project. Name it "Competitor Intelligence".
  2. Click Add content. Paste the contents of components/02-context-pack/competitor-a-notes.txt. Title it "Competitor A — Release Notes".
  3. Add a second document: paste components/02-context-pack/competitor-b-notes.txt. Title it "Competitor B — Release Notes".
  4. Add a third document: paste components/02-context-pack/our-product-summary.txt. Title it "Our Product — Current Features".
  5. Set a Project System Prompt: "You are a competitive intelligence analyst. Always cite the specific competitor document and section when making claims. Never invent features or pricing not present in the provided materials."
  6. Save the project.

Keeping intelligence fresh

Update competitor documents monthly (or when you spot a release). Edit the Project Knowledge document in-place — all future conversations automatically use the updated version. Date-stamp each document title (e.g. "Competitor A — Release Notes — June 2025") to track freshness.

Component 2

Prompt Library — Three Analysis Prompts

Save each prompt below to your Claude prompt library. Run them in sequence inside the Competitor Intelligence project during each analysis session.

Prompt 1: Research Extraction

Save as: "Intel — Extract Research"

Using the competitor documents in this project, extract the following for each competitor: 1. New or updated features (released in the last 90 days) 2. Current pricing tiers and key price points 3. Primary marketing messages / positioning language Format your output as three sections, one per competitor. Cite the document and section for each claim. If a piece of information is not available in the documents, say "not found in source." Do not invent or infer anything not explicitly stated in the materials.

Prompt 2: Gap Analysis

Save as: "Intel — Gap Analysis"

Based on the research extraction above and the "Our Product — Current Features" document in this project, produce a structured gap analysis. Output format: ## Feature Gap Table | Feature | Competitor A | Competitor B | Us | Priority Gap? | |---------|-------------|-------------|-----|--------------| [fill in rows] ## Pricing Comparison [one paragraph comparing pricing positioning] ## Messaging Opportunities [bullet list: 3–5 messaging angles where we have an advantage or where competitors are weak] ## Recommended Actions [numbered list: top 3 actions for product/marketing based on this analysis] Base every claim on the documents. Mark anything uncertain with (unconfirmed).

Prompt 3: Report Assembly

Save as: "Intel — Assemble Report"

Compile everything from this conversation into a final competitor intelligence report. Format as a clean Markdown document suitable for Notion or Confluence: # Competitor Intelligence Report **Date:** [today's date] **Competitors analysed:** [list] **Sources:** [document titles from project knowledge] ## Executive Summary (3 bullet points) ## Feature Gap Analysis [table from previous step] ## Pricing & Packaging [pricing comparison] ## Messaging Opportunities [list from previous step] ## Recommended Actions [numbered list] ## Source Notes [brief note on each document's date and coverage] Create this as an Artifact so I can copy it directly.
Component 3

Automation — Cowork Weekly Intelligence Session

Run a structured Cowork session inside the Competitor Intelligence project each week. The three prompts from your library handle the analysis in sequence.

Weekly workflow (15–20 minutes)

  1. Open the Competitor Intelligence Claude Project. Start a new conversation titled with today's date (e.g. "Intel Session — 28 June 2025").
  2. If there are new competitor releases this week, update the relevant Project Knowledge document before starting. (Edit → paste new content → save.)
  3. Apply the "Intel — Extract Research" prompt. Review the output — correct any misattributions before proceeding.
  4. Apply the "Intel — Gap Analysis" prompt. Expand any rows in the feature table that need more detail by asking follow-up questions in the same conversation.
  5. Apply the "Intel — Assemble Report" prompt. Claude will produce a formatted Artifact.
  6. Copy the Artifact and paste it into your team's Notion/Confluence page or share as a Slack message. The report is ready.

Optional: add a NotebookLM layer

For deeper research (e.g. analysing full product pages, PDFs, or YouTube demos), upload competitor materials to NotebookLM first and use its summary as your extraction input — then paste that summary into your Claude Project for gap analysis.

Putting It Together — End-to-End Workflow

Complete workflow for a competitive intelligence analyst using Claude.ai:

  1. Monthly setup: Update competitor documents in Project Knowledge with fresh release notes, pricing pages, and blog posts (paste as text).
  2. Weekly session: Open the project → apply Extract Research prompt → verify output.
  3. Analysis: Apply Gap Analysis prompt → review the feature table → ask Claude to elaborate on any row.
  4. Report: Apply Report Assembly prompt → Claude creates a formatted Artifact.
  5. Share: Copy Artifact to Notion/Confluence or paste into Slack. Total time per session: 15–20 minutes.
  6. Iteration: Over multiple sessions, your project knowledge builds a running history of competitor evolution — ask Claude to compare this week's gaps to last week's at any time.
