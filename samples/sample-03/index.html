Sample 03 · Capstone

Hiring Co-Pilot

Inconsistent screening slows hiring — solved with a grounded Claude Project, a standardised screening prompt library, and a Cowork comparison report workflow.

No API key needed. This capstone runs entirely inside your Claude.ai subscription — role context and company values loaded as Project Knowledge, screening prompts saved to your library, candidate comparison assembled via Cowork. No Python required.

The Problem & Solution Overview

The problem: Every interviewer asks different questions, evaluates candidates differently, and writes inconsistent notes. By the time the hiring panel meets, comparing candidates feels subjective — the person who interviewed best last week is remembered most vividly, not the one who scored highest on the rubric.

The solution: A Claude-powered hiring co-pilot where every interviewer uses the same question generator (grounded in role + values), the same scoring prompt, and the same note-compilation template — so the final comparison report is objective and repeatable.

Grounded Assistant (HO4) Screening Prompt Library (HO2) Automation / Comparison Report (HO5-6)

Output: consistent interview questions per role, standardised candidate scorecards, and a single comparison Artifact at the end of each hiring cycle — all generated in Claude.ai with no tools other than your subscription.

Component 1

Grounding — Set Up the Role Project

Create one Claude Project per open role. Load the role context as Project Knowledge so every interviewer and every prompt is grounded in the same criteria.

Steps

  1. Go to claude.ai → Projects → New Project. Name it after the role: e.g. "Hiring — Senior Product Designer".
  2. Add Project Knowledge document 1: paste components/02-context-pack/role-description.txt. Title it "Role Description".
  3. Add Project Knowledge document 2: paste components/02-context-pack/company-values.txt. Title it "Company Values".
  4. Add Project Knowledge document 3: paste components/02-context-pack/scoring-rubric.txt. Title it "Scoring Rubric".
  5. Set the Project System Prompt: "You are a hiring co-pilot. Every question you generate must map to one of the company values or role requirements in the project knowledge. Every score you assign must cite the scoring rubric. Never add criteria not present in the provided documents."
  6. Share the project link with all interviewers on the panel. Each opens the same project — same grounding, consistent criteria.
Component 2

Prompt Library — Three Screening Prompts

Save all three prompts to your Claude prompt library. Interviewers use Prompt 1 before their call, Prompt 2 after their call, and the hiring manager uses Prompt 3 to compile the final comparison.

Prompt 1: Screening Question Generator

Save as: "Hiring — Generate Questions"

Using the Role Description and Company Values in this project, generate a structured interview question set for a [INTERVIEW STAGE: e.g. first screen / technical / values] interview. Structure: - 4 questions tied to the role requirements - 1 question per company value (use the exact value names from the document) - 2 situational questions (past experience format: "Tell me about a time when...") For each question: - State which rubric dimension it evaluates - Suggest 2–3 follow-up probes Do not include any criteria not in the project knowledge documents.

Prompt 2: Candidate Scorer

Save as: "Hiring — Score Candidate"

Using the Scoring Rubric in this project, score the following candidate based on their interview transcript or notes. Candidate name: [NAME] Interview stage: [STAGE] Paste transcript or interviewer notes below: --- [PASTE NOTES OR TRANSCRIPT HERE] --- Output format: ## [Candidate Name] — Scorecard | Dimension | Score (1–5) | Evidence from Notes | |-----------|------------|---------------------| [one row per rubric dimension] **Overall score:** [average to 1 decimal] **Summary:** [2–3 sentences: strengths, gaps, recommendation] Base every score on the rubric. If a dimension was not assessed, mark as "N/A — not assessed in this interview."

Prompt 3: Note Compiler

Save as: "Hiring — Compile Notes"

Compile all candidate scorecards from this conversation into a structured comparison report for the hiring panel. Output as a Claude Artifact: # Candidate Comparison — [Role Name] **Date:** [today] **Stage:** [e.g. Final Round] **Candidates evaluated:** [list names] ## Score Summary Table | Candidate | [Dimension 1] | [Dimension 2] | [Dimension 3] | Overall | |-----------|--------------|--------------|--------------|---------| [fill from scorecards — use exact rubric dimension names from project knowledge] ## Individual Summaries [One paragraph per candidate: top strength, key gap, recommendation] ## Hiring Panel Recommendation [Based only on the scores above, rank candidates and state a recommendation with rationale] Important: base all scores and recommendations strictly on the scorecards. Do not introduce new criteria.
Component 3

Automation — Cowork Candidate Compilation

The hiring manager runs one Cowork session after all interviews are complete to compile the comparison report — no spreadsheets needed.

Post-interview compilation steps

  1. Open the hiring project. Start a new conversation titled "Final Comparison — [Date]".
  2. For each candidate, apply the "Hiring — Score Candidate" prompt. Paste that interviewer's notes or the interview transcript (from a recording tool).
  3. After all candidates are scored in the same conversation, apply the "Hiring — Compile Notes" prompt. Claude reads all the scorecards from the conversation history and generates the comparison table.
  4. Review the Artifact. If any score seems off, ask Claude: "Explain the score for [candidate] on [dimension]" — it will cite the specific line from their notes.
  5. Share the Artifact link (or copy) with the hiring panel before the decision meeting.

Putting It Together — End-to-End Hiring Workflow

Complete workflow from job post to hiring decision using Claude.ai:

  1. Role opens: Create the Hiring project, load role description + values + rubric as Project Knowledge, share the project with the panel.
  2. Before each interview: Interviewer opens the project → applies the "Generate Questions" prompt specifying their interview stage → gets a customised, rubric-aligned question set.
  3. After each interview: Interviewer pastes their raw notes into the project → applies "Score Candidate" → gets a structured scorecard saved in the conversation.
  4. After all interviews: Hiring manager applies "Compile Notes" in a session that includes all scorecards → gets a comparison Artifact.
  5. Decision meeting: Panel reviews the Artifact — scoring is consistent, evidence-based, and fully documented. Decision is faster and more defensible.
